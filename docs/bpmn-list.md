### **Верхнеуровневые бизнес-процессы Zapuskalka**

#### **A. Процессы взаимодействия с пользователем (User Journey)**

1.  **Регистрация и онбординг**
    *   BP-USER-001: Процесс регистрации нового пользователя
    *   BP-USER-002: Процесс входа в систему
    *   BP-USER-003: Процесс восстановления пароля
    *   BP-USER-004: Процесс первоначальной настройки клиента (выбор языка, создание первой библиотеки)

2.  **Обнаружение и приобретение игр**
    *   BP-STORE-001: Процесс просмотра каталога игр (с фильтрацией и поиском)
    *   BP-STORE-002: Процесс просмотра страницы игры
    *   BP-STORE-003: Процесс "покупки" бесплатной игры (добавления в библиотеку) *(для MVP)*
    *   BP-STORE-004: Процесс покупки платной игры *(для Release 3.0)*

3.  **Управление библиотекой и играми**
    *   BP-LIB-001: Процесс установки игры
    *   BP-LIB-002: Процесс запуска игры
    *   BP-LIB-003: Процесс удаления игры
    *   BP-LIB-004: Процесс проверки и установки обновлений игры

4.  **Управление загрузками и хранилищем**
    *   BP-DL-001: Процесс управления загрузками (пауза, возобновление, отмена)
    *   BP-STORAGE-001: Процесс создания и настройки новой библиотеки/хранилища
    *   BP-STORAGE-002: Процесс изменения пути установки по умолчанию

#### **B. Процессы разработчика (Developer Portal)**

5.  **Публикация и управление играми**
    *   BP-DEV-001: Процесс регистрации разработчика
    *   BP-DEV-002: Процесс первоначальной публикации игры (загрузка билда, метаданных, скриншотов)
    *   BP-DEV-003: Процесс обновления опубликованной игры (загрузка нового билда, патч-ноутов)
    *   BP-DEV-004: Процесс управления страницей игры (редактирование описания, добавление скриншотов)

#### **C. Операционные и системные процессы (Backend & Core)**

6.  **Системные процессы клиента**
    *   BP-SYS-001: Процесс запуска и инициализации клиента
    *   BP-SYS-002: Процесс автоматического обновления клиента
    *   BP-SYS-003: Процесс обработки сетевых ошибок и повторных попыток запросов

7.  **Процессы взаимодействия Клиент-Сервер**
    *   BP-API-001: Процесс аутентификации и валидации сессии
    *   BP-API-002: Процесс синхронизации состояния библиотеки пользователя
    *   BP-API-003: Процесс загрузки файла игры (CDN/Storage)

#### **D. Бизнес-процессы (Post-MVP)**

8.  **Монетизация и платежи**
    *   BP-PAY-001: Процесс обработки платежа (единоразовая покупка)
    *   BP-PAY-002: Процесс обработки модели "Pay What You Want"
    *   BP-PAY-003: Процесс генерации, распространения и активации ключей
    *   BP-PAY-004: Процесс выплаты средств разработчику

9.  **Социальные функции**
    *   BP-SOC-001: Процесс управления списком друзей (добавление, принятие заявок)
    *   BP-SOC-002: Процесс обмена сообщениями
    *   BP-SOC-003: Процесс отображения статуса "В сети/В игре"

---

**Рекомендации по моделированию:**

1.  **Начните с "плавательных дорожек" (Swimlanes):** Пользователь, Клиент Zapuskalka, Бэкенд-API, Сервис хранения файлов, Микросервис оплаты (отдельно).
2.  **Приоритет для MVP:** Сначала смоделируйте процессы из групп **A, B и C**. Это ядро вашего продукта.
3.  **Ключевые процессы для старта:**
    *   **BP-USER-001 + BP-API-001** (полный цикл аутентификации)
    *   **BP-LIB-001 + BP-API-003 + BP-DL-001** (ядро продукта: установка и управление загрузками)
    *   **BP-DEV-002** (публикация игры — критично для наполнения платформы)

Этот список дает исчерпывающее представление о всех бизнес-процессах системы и является отличной основой для создания набора согласованных BPMN-диаграмм.
