### **Верхнеуровневые бизнес-процессы Zapuskalka**

#### **A. Процессы взаимодействия с пользователем (User Journey / Release 1.0)**

1.  **Аутентификация и профиль (авторегистрация)**
    - BP-USER-001: Процесс входа через OTP (email/код) с автоматической регистрацией при первом входе
    - BP-USER-002: Процесс входа через OAuth (Github/Twitch) с автоматической регистрацией при первом входе
    - BP-USER-003: Процесс восстановления доступа (повторный OTP / смена способа входа)
    - BP-USER-004: Процесс первоначальной настройки клиента (выбор языка, создание первой библиотеки)
    - BP-USER-005: Процесс настройки и отображения публичной страницы профиля (ник, аватар)

2.  **Обнаружение и приобретение игр (Магазин / "Все игры")**
    - BP-STORE-001: Процесс просмотра каталога игр (с фильтрацией и поиском)
    - BP-STORE-002: Процесс просмотра страницы игры
    - BP-STORE-003: Процесс "покупки" бесплатной игры (добавления в библиотеку) _(для MVP)_
    - BP-STORE-004: Процесс покупки платной игры _(для Release 3.0)_
    - BP-STORE-005: Процесс поиска по каталогу (по названию)
    - BP-STORE-006: Процесс фильтрации каталога (жанры, теги, платформы)
    - BP-STORE-007: Процесс отображения статуса владения ("В библиотеке", переход в "Мои игры")

3.  **Управление библиотекой и играми ("Мои игры")**
    - BP-LIB-001: Процесс установки игры
    - BP-LIB-002: Процесс запуска игры
    - BP-LIB-003: Процесс удаления игры
    - BP-LIB-004: Процесс проверки и установки обновлений игры
    - BP-LIB-005: Процесс поиска и фильтрации внутри библиотеки (жанры, теги, установленный статус)
    - BP-LIB-006: Процесс работы со страницей игры в библиотеке (кнопки Установить/Запустить/Обновить, контекстное меню)
    - BP-LIB-007: Процесс управления владельцем игры (Удалить с устройства vs удалить из аккаунта)

4.  **Управление загрузками и хранилищем**
    - BP-DL-001: Процесс управления загрузками (пауза, возобновление, отмена)
    - BP-STORAGE-001: Процесс создания и настройки новой библиотеки/хранилища
    - BP-STORAGE-002: Процесс изменения пути установки по умолчанию
    - BP-STORAGE-003: Процесс выбора библиотеки по умолчанию и назначения игры в конкретную библиотеку
    - BP-DL-002: Процесс фоновой загрузки с отображением прогресса (скорость, ETA, объем)

#### **B. Процессы разработчика (Developer Portal / Release 1.0)**

5.  **Публикация и управление играми**
    - BP-DEV-001: Процесс регистрации разработчика
    - BP-DEV-002: Процесс первоначальной публикации игры (загрузка билда, метаданных, скриншотов)
    - BP-DEV-003: Процесс обновления опубликованной игры (загрузка нового билда, патч-ноутов)
    - BP-DEV-004: Процесс управления страницей игры (редактирование описания, добавление скриншотов)
    - BP-DEV-005: Процесс указания версии, системных требований и описания (валидация перед публикацией)

#### **C. Операционные и системные процессы (Backend & Core / Release 1.0)**

6.  **Системные процессы клиента**
    - BP-SYS-001: Процесс запуска и инициализации клиента
    - BP-SYS-002: Процесс автоматического обновления клиента
    - BP-SYS-003: Процесс обработки сетевых ошибок и повторных попыток запросов
    - BP-SYS-004: Процесс проверки целостности установленных файлов клиента

7.  **Процессы взаимодействия Клиент-Сервер**
    - BP-API-001: Процесс аутентификации и валидации сессии
    - BP-API-002: Процесс синхронизации состояния библиотеки пользователя
    - BP-API-003: Процесс загрузки файла игры (CDN/Storage)
    - BP-API-004: Процесс проверки и доставки обновлений игр

#### **D. Бизнес-процессы (Post-MVP: Release 2.0 и 3.0)**

8.  **Монетизация и платежи (Release 3.0)**
    - BP-PAY-001: Процесс обработки платежа (единоразовая покупка)
    - BP-PAY-002: Процесс обработки модели "Pay What You Want"
    - BP-PAY-003: Процесс генерации, распространения и активации ключей
    - BP-PAY-004: Процесс выплаты средств разработчику
    - BP-PAY-005: Процесс интеграции проприетарного микросервиса оплаты с основным бэкендом

9.  **Социальные функции и расширенные возможности (Release 2.0)**
    - BP-SOC-001: Процесс управления списком друзей (добавление, принятие заявок)
    - BP-SOC-002: Процесс обмена сообщениями (1 на 1)
    - BP-SOC-003: Процесс отображения статуса "В сети/В игре"
    - BP-SOC-004: Процесс группового чата
    - BP-DL-003: Процесс расширенного планирования загрузок (ограничение скорости, расписание)
    - BP-DL-004: Процесс проверки целостности файлов игры
    - BP-DEV-006: Процесс управления страницей разработчика и лентой новостей по играм
    - BP-DEV-007: Процесс предоставления расширенной аналитики разработчику

---

**Рекомендации по моделированию:**

1.  **Начните с "плавательных дорожек" (Swimlanes):** Пользователь, Клиент Zapuskalka, Бэкенд-API (PocketBase), Сервис хранения файлов (S3), Микросервис оплаты (отдельно), Разработчик.
2.  **Приоритет для MVP:** Сначала смоделируйте процессы из групп **A, B и C** — это перечень требований Release 1.0 по PRODUCT.md.
3.  **Ключевые процессы для старта (соответствуют целям O1/O2 PRD):**
    - **BP-USER-001/002 + BP-API-001** (полный цикл аутентификации, включая автоматическую регистрацию и управление сессией)
    - **BP-STORE-001/005 + BP-LIB-001 + BP-API-003 + BP-DL-001/002** (обнаружение → установка → запуск с управлением загрузкой)
    - **BP-DEV-002 + BP-DEV-005** (публикация игры с обязательными метаданными)
4.  **Расширения Release 2.0 и 3.0:** моделируйте группы **D.8** и **D.9** после стабилизации MVP, учитывая дополнительные акторы (микросервис оплаты, социальные сервисы).
